### 1. Исправить бесконечную перезагрузку страницы турнира
**Проблема**: После регистрации на турнир страница бесконечно перезагружается
**Причина**: Скорее всего хук useEffect на фронтенде вызывается многократно
**Файлы**: `frontend/src/app/tournaments/[id]/page.tsx`

### 2. Убрать поле "Название команды" из регистрации на турнир
**Проблема**: При регистрации просят ввести название команды - это лишнее
**Решение**: 
- Удалить поле `teamName` из формы регистрации
- Сделать поле `teamName` необязательным в backend
- Логика: если турнир клановый - участник должен выбрать свой клан автоматически
**Файлы**: 
- Backend: `participants.service.ts`, `participant.entity.ts`
- Frontend: компонент регистрации на турнир

## 🟢 НОВЫЕ ФИЧИ (БОЛЬШИЕ ИЗМЕНЕНИЯ)

### 5. Переделать систему подписок в "Запросы в друзья"
**Текущая система**: Односторонняя подписка (как Instagram)
**Новая система**: Двухсторонняя дружба (как ВКонтакте)

**Изменения**:
1. Создать таблицу `friend_requests` (запросы в друзья)
2. Изменить таблицу `follows` → `friendships` (принятые друзья)
3. Добавить статусы: `PENDING`, `ACCEPTED`, `REJECTED`
4. API endpoints:
   - `POST /users/:id/friend-request` - отправить запрос
   - `GET /users/me/friend-requests` - получить запросы
   - `POST /users/me/friend-requests/:id/accept` - принять
   - `POST /users/me/friend-requests/:id/reject` - отклонить
   - `DELETE /users/:id/unfriend` - удалить из друзей
   - `GET /users/:id/friends` - список друзей

**Файлы**:
- Backend: создать `friend-request.entity.ts`, обновить `follow.entity.ts`
- Backend: создать `friends` module
- Frontend: создать страницу запросов в друзья
- Frontend: обновить профиль (кнопка "Добавить в друзья")

### 6. Создать чат как в Telegram
**Дизайн**: 
- Левая часть: список чатов с превью последнего сообщения
- Правая часть: окно чата с сообщениями

**Технологии**: WebSocket (Socket.io)

**Backend изменения**:
1. Установить `@nestjs/websockets`, `socket.io`
2. Создать `chat` module
3. Создать entities:
   - `chat-room.entity.ts` - приватные чаты между двумя друзьями
   - `message.entity.ts` - сообщения
4. Gateway для WebSocket соединений
5. API endpoints:
   - `GET /chat/rooms` - список чатов
   - `GET /chat/rooms/:id/messages` - сообщения чата
   - `POST /chat/rooms/:id/messages` - отправить сообщение

**Frontend изменения**:
1. Создать страницу `/chat`
2. Layout: 2 колонки (список чатов + окно чата)
3. Подключить Socket.io client
4. Реал-тайм обновление сообщений

**Дизайн компонентов**:
```
┌─────────────────────────────────────┐
│ Header (Чаты)                       │
├──────────────┬──────────────────────┤
│ ChatsList    │ ChatWindow           │
│              │                      │
│ Friend 1 >   │ Messages             │
│ Friend 2     │                      │
│ Friend 3     │                      │
│              │ Input field          │
└──────────────┴──────────────────────┘
```
